2 Authentication and Web Portal (Sequence Diagram)

                    sequenceDiagram
                        autonumber
                        actor U as 👤 User
                        participant WP as 🌐 Web Portal (AFSMS)
                        participant SSO as 🔐 SSO Subsystem
                        participant DB as 🗄️ Database

                        U->>WP: Access /login
                        WP->>SSO: Redirect (Token Request)
                        SSO-->>U: Display institutional login page
                        
                        Note over U,SSO: The user enters their university credentials
                        U->>SSO: Enter ucv.ro credentials
                        
                        SSO-->>WP: Return valid token (SAML/OAuth)
                        WP->>DB: Query permissions based on ID
                        DB-->>WP: Return Role (e.g., Registrar)
                        WP-->>U: Display private interface (Dashboard)

5 Audit Logging and Rollback (Sequence Diagram)

                    sequenceDiagram
                        autonumber
                        actor Admin as 👤 Administrator
                        participant WP as 🌐 AFSMS Interface
                        participant DB as 🗄️ Database (DBMS)

                        Admin->>WP: Request Audit Logs view
                        WP->>DB: Extract activity history
                        DB-->>WP: Return data (Actor, Time, Action)
                        WP-->>Admin: Display audit logs table

                        rect rgb(241, 245, 249)
                        opt Point-in-time Rollback Scenario
                            Admin->>WP: Initiate Rollback to time T-1
                            WP->>DB: Command restoration (Backup / Log)
                            
                            activate DB
                            Note right of DB: DBMS cancels recent
critical operations
                            DB->>DB: Execute Rollback
                            DB-->>WP: Confirm complete restoration
                            deactivate DB
                            
                            WP->>DB: Save "Rollback executed" action in log
                            WP-->>Admin: Success message on screen
                        end
                        end
